# v1.1 云同步计划

本文档把 v1.1 可选云同步拆成可执行任务，并为每个任务定义验收证据。它复用 [安全实现设计](../technical/security-implementation-design.md) 第 8 节的同步密码学规则、[数据模型](../technical/data-model.md) 的 SyncObject 与 DeviceRecord、[技术架构](../technical/architecture.md) 第 6 节的云同步数据流，并对齐 [版本与权益计划](version-plan.md) 的 v1.1 退出标准与登录策略、[路线图](roadmap.md) 阶段 3，以及 [测试计划](../testing/test-plan.md) 的 TP-201 至 TP-206。

本文档服从 [产品决策记录：主动选择式私密输入](product-decision-record.md)：云同步是可选增强，本地使用永不依赖云；登录只用于云服务，绝不作为本地使用的前置；主动选择仍是核心心智模型；平台凭据与自动填充是兼容增强，不是产品主路径。

## 1. 目标

v1.1 增加基于付费和登录的云服务，同时不削弱离线优先行为。它要让用户能够：

> 在自愿登录后启用端到端加密云同步，在多台设备间收敛保险库，处理并发编辑冲突，并查看版本历史与恢复；同时云端只存储密文，退出登录或离线时本地保险库仍完全可用，同步失败不阻断本地使用。

## 2. 平台与范围

首平台为 Android。iOS 和 HarmonyOS 在 v1.1 不构建实现。

v1.1 在范围内：

- 用户账号与身份。
- 端到端加密云同步。
- 设备列表与注册。
- 同步冲突检测与解决。
- 版本历史与恢复。
- 订阅与权益校验。

v1.1 不在范围内：

- 改变离线优先行为或要求登录才能本地使用。
- 多保险库与高级模板。
- iOS 与 HarmonyOS 实现构建。

## 3. 调研转译

本轮在同步复制模型与冲突检测上做了调研；端到端加密的服务端零知识属性复用安全实现设计。

### 3.1 乐观复制与离线优先

- 同步有两种复制模型：强一致复制需要副本间协调，断网时无法在本地修改；乐观复制允许任意副本独立修改，即使离线，冲突在副本通信时再解决。
- 本产品离线优先，应采用乐观复制：用户离线编辑，联网后再同步。

### 3.2 冲突检测（版本向量）

- 版本向量用于判断两次更新是先后发生还是并发发生；并发更新即潜在冲突。
- 数据模型的 SyncObject 已带 baseVersion 与 version，可用于检测并发编辑：两台设备从同一基线编辑即判定为并发冲突。

### 3.3 冲突解决（CRDT 与无丢失）

- CRDT 可在乐观复制下自动合并到一致状态，无需人工干预，但会增加复杂度。
- 对保险库，优先采用条目和字段级粒度降低冲突，对真正冲突提供用户可见的解决，并以版本历史避免静默丢失；CRDT 式自动合并可作为后续可选项。

### 3.4 复用的安全规则

- 同步载荷使用独立于本地内容密钥的域分离同步密钥，带认证，先校验后使用：见 [安全实现设计](../technical/security-implementation-design.md) 第 8 节。
- 服务端只存储密文，不接收主密码与明文：见 [安全模型](../technical/security-model.md) 第 7 节。

## 4. 同步架构与信任边界

```text
本地保险库变更
  -> 核心创建端到端加密同步载荷（域分离同步密钥）
  -> 平台上传密文与版本元数据
  -> 云端存储带版本的加密对象
  -> 另一台设备下载密文
  -> 核心本地校验、解密并合并
```

信任边界：

- 登录只用于账号身份、设备管理、权益和云对象访问，不用于解密保险库。
- 主密码仍是根秘密，留在客户端；服务端绝不接收主密码或保险库明文。
- 云账号密码重置不得解密保险库数据。
- 云端只存储密文；冲突与版本元数据可对服务端可见，但内容保持加密。
- 云同步是可选功能；退出登录或离线时本地保险库完全可用。

## 5. 同步与冲突模型

- 采用乐观复制：任意设备可离线编辑，联网后同步。
- 冲突检测基于版本：用 baseVersion 与 version 判定并发编辑。
- 冲突解决优先条目和字段级粒度，减少真实冲突面。
- 真正冲突提供用户可见的解决，不静默覆盖。
- 版本历史保留变更，避免数据丢失，并支持恢复。
- 同步对象采用追加式版本与认证载荷，导入合并失败可回滚。

## 6. 构建块

```text
v1.1 云同步
  -> 账号与身份
  -> 端到端加密同步
  -> 设备管理
  -> 冲突检测与解决
  -> 版本历史与恢复
  -> 订阅与权益
  -> 离线优先保障
```

## 7. 任务分解

每个任务定义目的、交付物和验收证据。

### SYNC-001 账号与身份

- 目的：提供仅用于云服务的登录。
- 交付物：账号注册与登录、会话管理、登录仅作为云服务前置。
- 验收证据：登录只解锁云功能；本地创建、解锁、编辑、键盘输入不要求登录；登录不参与保险库解密。

### SYNC-002 端到端加密同步

- 目的：在云端只存储密文。
- 交付物：域分离同步密钥派生、同步载荷加密与认证、密文上传与下载。
- 验收证据：服务端载荷不含明文字段；同步密钥独立于本地内容密钥；服务端不接收主密码。

### SYNC-003 设备列表与注册

- 目的：管理参与同步的设备。
- 交付物：基于 DeviceRecord 的设备注册、设备列表、撤销设备。
- 验收证据：设备可注册并显示在列表；撤销后不再获得新密文访问；本地单机使用不需要设备记录。

### SYNC-004 同步冲突检测

- 目的：识别并发编辑。
- 交付物：基于 baseVersion 与 version 的并发检测、追加式版本。
- 验收证据：两台设备从同一基线编辑被判为冲突；先后编辑不误报为冲突；离线编辑联网后正确进入检测。

### SYNC-005 冲突解决与无丢失合并

- 目的：在不丢数据的前提下解决冲突。
- 交付物：条目和字段级合并、用户可见的冲突解决、默认不静默覆盖。
- 验收证据：真实冲突由用户决策；合并不静默丢失任意一方数据；解决过程可回滚。

### SYNC-006 版本历史与恢复

- 目的：提供变更历史与恢复能力。
- 交付物：版本历史记录、按版本恢复、历史只含密文。
- 验收证据：可查看并恢复历史版本；历史在服务端以密文存储；恢复不破坏当前有效保险库。

### SYNC-007 订阅与权益校验

- 目的：以权益区分付费云功能。
- 交付物：订阅状态、权益校验、权益仅约束云功能。
- 验收证据：权益只影响云功能可用性；权益异常不阻断本地使用；权益校验不要求暴露保险库内容。

### SYNC-008 离线优先保障

- 目的：保证本地使用永不依赖云。
- 交付物：退出登录保留本地保险库、离线本地可用、同步失败不阻断本地。
- 验收证据：退出登录后本地保险库仍可用；离线时核心功能可用；同步失败时本地编辑与键盘输入不受影响。

## 9. 退出标准

v1.1 完成需同时满足版本计划退出标准和同步验证：

- 云端只存储密文。
- 用户退出登录或离线时，本地保险库仍可使用。
- 同步失败不会阻断本地使用。
- 并发冲突可检测并由用户解决，无数据丢失。
- 登录只用于云服务，不参与本地使用与保险库解密。

## 10. 风险与兜底

| 风险 | 影响 | 兜底 |
| --- | --- | --- |
| 冲突合并设计不当 | 数据丢失。 | 条目级粒度、用户可见解决、版本历史、可回滚。 |
| 登录被误设为本地前置 | 违反离线优先与登录策略。 | 登录仅约束云功能；本地路径不经过登录。 |
| 服务端元数据泄露内容 | 削弱零知识属性。 | 只上传密文；元数据最小化；内容始终加密。 |
| 同步中断或失败 | 阻断本地使用。 | 同步失败降级为本地可用，稍后重试。 |
| 权益异常影响本地 | 本地被锁。 | 权益只影响云功能，不触及本地保险库。 |

## 11. 不在 v1.1 范围

- 改变离线优先行为或要求登录才能本地使用。
- 多保险库与高级模板。
- 平台自动填充、Credential Provider、TOTP 自动填充。
- iOS 与 HarmonyOS 实现构建。

## 12. 下一步

v1.1 完成后，版本与权益计划中的发布阶段序列（v0.1 至 v1.1）已全部形成可执行计划。后续方向建议从路线图阶段 4 高级输入展开，或回到 iOS 与 HarmonyOS 的平台实现验证，按需要再拆分为独立计划。
