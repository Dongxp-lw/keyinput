# v0.2 离线 MVP 计划

本文档把 v0.2 本地保险库 MVP 拆成可执行任务，并为每个任务定义验收证据。它承接 [v0.1 原型计划](v0.1-prototype-plan.md) 验证过的键盘插入能力，复用 [安全实现设计](../technical/security-implementation-design.md) 的密码学决策，并对齐 [版本与权益计划](version-plan.md) 的 v0.2 退出标准、[信息架构设计](information-architecture.md)、[核心交互设计](interaction-design.md)，以及 [测试计划](../testing/test-plan.md) 的 TP-101 至 TP-109。

本文档服从 [产品决策记录：主动选择式私密输入](product-decision-record.md)：主动选择是核心心智模型；生物识别是便利机制，不替代主密码；平台凭据与自动填充是标准登录场景的兼容增强，不是产品主路径。

## 1. 目标

v0.2 交付第一个真正可用的离线产品。在 v0.1 证明键盘可以主动选择并插入字段之后，v0.2 要让用户能够：

> 不登录、不联网地创建本地加密保险库，用主密码和生物识别解锁，管理条目和字段，从安全键盘主动选择并插入真实保险库中的值，生成密码和基础 TOTP，并通过加密包导出和恢复保险库。

## 2. 平台与范围

首平台延续 Android（理由见 [平台能力设计](platform-capability-design.md) 第 7 节）。iOS 和 HarmonyOS 在 v0.2 不构建实现，相关平台限制继续记录。

v0.2 在范围内：

- 本地加密保险库，按 [安全实现设计](../technical/security-implementation-design.md) 的格式与密钥层级。
- 主密码设置与解锁。
- 生物识别或设备凭据解锁。
- 条目和字段的增删改查。
- 安全键盘从真实保险库主动选择并插入字段。
- 密码生成器。
- 基础 TOTP 生成。
- 加密导出与导入。
- 带自动清除的剪贴板备选方案。

v0.2 不在范围内：

- 云账号、同步、订阅。
- 跨设备迁移（属于 v0.3）。
- 多保险库与高级模板（属于 Pro）。
- iOS 与 HarmonyOS 实现构建。

## 3. 调研转译

本轮在 TOTP 和密码生成两个 v0.2 新主题上做了权威调研；保险库加密和生物识别绑定直接复用安全实现设计，不再重复决策。

### 3.1 TOTP（RFC 6238 / RFC 4226）

- 算法：`TOTP = HOTP(K, T)`，其中 `T = floor((当前 Unix 时间 - T0) / X)`。
- 默认参数：时间步 X = 30 秒，T0 = 0（Unix 纪元）。
- 位数：至少 6 位，支持 8 位。
- 哈希：默认 HMAC-SHA-1 兼容性最好，可选 HMAC-SHA-256 / HMAC-SHA-512。
- 必须支持 2038 年后大于 32 位整数的时间步值。
- 种子：至少 128-bit，RFC 4226 推荐 160-bit。
- 产品定位是生成方（prover），验证码依赖设备时间，时钟漂移会影响结果。
- 用 RFC 6238 附录 B 的测试向量做已知答案测试，例如密钥 ASCII `12345678901234567890`、X=30、T0=0 时，59 秒对应 SHA-1 8 位验证码 `94287082`。
- 常见的 `otpauth://` Key URI 是事实标准而非 RFC 6238 的一部分，是否在 v0.2 支持导入待确认。

### 3.2 密码生成器

- 使用 CSPRNG 生成，不使用非加密安全随机源。
- 长度和字符集可配置。
- 把随机值映射到字符集时避免取模偏置，例如采用拒绝采样。
- 可选排除易混淆字符。
- 生成的密码不写入日志。

### 3.3 复用的安全决策

- 保险库加密、KDF、密钥层级、文件格式、解锁会话、内存与日志卫生：见 [安全实现设计](../technical/security-implementation-design.md)。
- 生物识别绑定平台密钥库：见同文档第 5、6 节。
- TOTP 种子作为高敏感字段加密存储、不记日志：见 [安全模型](../technical/security-model.md) 与数据模型。

## 4. MVP 构建块

```text
v0.2 离线 MVP
  -> 保险库核心与加密
  -> 主应用
  -> 安全键盘
  -> 生成器与 TOTP
  -> 导入导出
  -> 验证工具
```

- 保险库核心与加密：实现安全实现设计定义的保险库格式、KDF、信封密钥和 AEAD。
- 主应用：保险库创建、解锁、条目管理、生成器与导入导出入口。
- 安全键盘：从真实保险库解锁、搜索、主动选择并插入字段。
- 生成器与 TOTP：密码生成器与基础 TOTP 生成。
- 导入导出：加密导出包的生成、校验与恢复。
- 验证工具：单元测试、instrumented 测试、UI Automator、日志扫描、离线验证。

## 5. 任务分解

每个任务定义目的、交付物和验收证据。

### MVP-001 保险库加密核心

- 目的：实现本地加密保险库的读写。
- 交付物：按安全实现设计的保险库文件格式、Argon2id 派生 KEK、随机 DEK 信封包装、XChaCha20-Poly1305 内容加密、头部作为 AAD 认证。
- 验收证据：保险库可加密保存并解密读取；篡改头部或密文导致解密失败；已知答案测试覆盖 KDF 与 AEAD。

### MVP-002 主密码设置与解锁

- 目的：以主密码作为根秘密保护保险库。
- 交付物：主密码创建、强度提示、解锁流程、错误计数。
- 验收证据：正确主密码可解锁；错误主密码解包失败且不泄露区分信息；KDF 参数随头部保存并被读取。

### MVP-003 生物识别解锁

- 目的：在主密码之外提供便利解锁。
- 交付物：基于平台密钥库的本地解锁密钥包装、BiometricPrompt 集成、关闭开关、新生物识别注册使密钥失效。
- 验收证据：生物识别可解锁；可关闭并回退主密码；新增生物识别注册后受保护密钥按设计失效；生物识别不替代主密码。

### MVP-004 条目与字段增删改查

- 目的：让用户管理保险库内容。
- 交付物：条目与字段的创建、编辑、删除、搜索，覆盖用户名、密码、TOTP、自定义字段、备注。
- 验收证据：条目可保存并可搜索；字段携带类型与敏感级别；删除与编辑持久生效；纯逻辑有单元测试。

### MVP-005 安全键盘真实保险库选择

- 目的：把 v0.1 的插入能力接到真实加密保险库。
- 交付物：键盘解锁交接、搜索、字段主动选择、`commitText()` 插入、最小短时会话授权。
- 验收证据：解锁后可从真实保险库选择并插入字段；锁定态不泄露条目与字段；密码字段不在键盘明文展示；只有用户点击字段才插入。

### MVP-006 密码生成器

- 目的：帮助用户创建强密码。
- 交付物：基于 CSPRNG 的生成器、可配置长度与字符集、避免取模偏置、可选排除易混淆字符。
- 验收证据：生成结果符合所选长度与字符集；统计上无明显偏置；生成值不写入日志。

### MVP-007 基础 TOTP 生成

- 目的：把 2FA 作为高敏感字段支持。
- 交付物：按 RFC 6238 实现 TOTP，默认 X=30、T0=0、6 位，支持 8 位与 SHA-1/256/512，支持 2038 年后时间步。
- 验收证据：RFC 6238 附录 B 测试向量全部通过；TOTP 种子加密存储且不记日志；验证码随时间步刷新。

### MVP-008 加密导出与导入

- 目的：让用户在本地备份和恢复保险库。
- 交付物：加密导出包生成、导入校验、失败回滚、版本与架构兼容性检查。
- 验收证据：导出包不能按明文读取；导入条目与源保险库一致；被篡改的导出包安全失败；导入失败不破坏现有保险库。

### MVP-009 剪贴板备选方案

- 目的：在键盘插入不可用时提供兜底。
- 交付物：复制动作、配置化自动清除、平台支持时标记敏感。
- 验收证据：复制后在配置超时内自动清除（平台允许时）；备选路径有文档说明。

### MVP-010 离线与隐私验证

- 目的：保持 v0.1 的离线与隐私保证。
- 交付物：不声明 INTERNET 权限、网络抓包验证、日志扫描、开发构建 StrictMode。
- 验收证据：无 INTERNET 权限；抓包无流量；日志不含主密码、密钥、字段值、TOTP 种子和生成的验证码。

## 7. 离线与隐私验证

- 离线主要保证：不声明 `android.permission.INTERNET`。
- 补充验证：网络安全配置关闭明文、网络抓包无流量、开发构建启用 StrictMode。
- 隐私验证：日志和崩溃报告不含主密码、密钥、字段值、TOTP 种子和生成的验证码。
- 篡改验证：保险库或导出包被篡改时解密或导入安全失败。

## 8. 退出标准

v0.2 完成需同时满足版本计划退出标准和 MVP 验证：

- 用户可以离线创建、使用、导出和恢复保险库。
- 敏感字段不会被写入日志。
- 核心加密测试和保险库迁移测试通过。
- 主密码与生物识别解锁可用，生物识别可关闭。
- 安全键盘从真实保险库主动选择并插入字段。
- TOTP 通过 RFC 6238 测试向量。
- 导入失败不破坏现有保险库。

## 9. 风险与兜底

| 风险 | 影响 | 兜底 |
| --- | --- | --- |
| 设备时钟漂移 | TOTP 验证码与服务端不一致。 | 依赖设备时间并提示用户校时；记录为生成方限制。 |
| 生物识别在部分设备不可用 | 解锁便利性下降。 | 始终保留主密码解锁路径。 |
| 密码字段限制键盘插入 | 部分场景无法插入。 | 提供剪贴板兜底并记录限制。 |
| Argon2id 参数过高拖慢低端设备 | 解锁卡顿。 | 按设备基准测试调参，低内存设备降级到 OWASP 下限档并记录。 |
| 导入损坏数据 | 数据丢失风险。 | 认证校验、版本校验、失败回滚不覆盖有效库。 |

## 10. 不在 v0.2 范围

- 云账号、同步、订阅。
- 跨设备迁移与配对（属于 v0.3）。
- 多保险库与高级模板。
- 平台自动填充、Credential Provider、TOTP 自动填充。
- iOS 与 HarmonyOS 实现构建。

## 11. 下一步

v0.2 完成并满足退出标准后，进入 v0.3 跨设备迁移。下一份文档建议是“v0.3 跨设备迁移计划”，把加密迁移包、本地配对或二维码迁移、迁移校验与可恢复导入、导入冲突处理拆成可执行任务与验收证据。
