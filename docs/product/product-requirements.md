# 产品需求

## 1. 产品定位

Private Input Vault 是一个离线优先的私密键盘和加密保险库。它帮助用户在应用和网站中主动选择并输入密码、用户名、2FA 验证码、账号字段、地址、备注以及可复用的私密文本片段。

本产品不把系统自动填充作为主要路径，也不猜测应该填入什么内容。用户从安全键盘或主应用中选择目标值。

## 2. 目标

- 让用户无需登录、无需网络即可创建和使用加密保险库。
- 提供安全键盘，用于由用户选择的私密输入。
- 支持密码、自定义字段、备注和 2FA/TOTP 值。
- 提供可选的跨设备加密迁移和同步能力。
- 同时支持第一方云服务和用户自主控制的迁移路径。
- 保持云存储端到端加密，确保服务端永远拿不到保险库明文数据。

## 3. 非目标

- 不把系统自动填充作为产品主路径。
- 不承诺钓鱼检测或网站真实性判断。
- 不在缺少用户操作时静默选择或填入敏感数据。
- 不要求用户登录账号后才能使用本地保险库。
- 默认不提供键盘词库或在线词库。
- 不提供服务端恢复保险库主密码的能力。

## 4. 目标用户

- 想要完全离线可用的密码与私密字段管理工具的用户。
- 偏好手动控制而不是自动填充的用户。
- 经常在不同应用中输入重复私密字段的用户。
- 希望在设备间迁移加密数据，同时不被某个云服务绑定的用户。

## 5. 核心用户流程

### 5.1 创建本地保险库

1. 用户首次打开应用。
2. 用户创建本地保险库和主密码。
3. 应用说明：如果没有备份或恢复配置，忘记主密码后无法恢复。
4. 用户可选择启用生物识别解锁，提升日常使用便利性。

### 5.2 保存条目

1. 用户创建保险库条目。
2. 用户添加用户名、密码、TOTP 种子、邮箱、地址、备注或自定义文本等字段。
3. 敏感字段在静态存储时加密。
4. 用户设置标签、收藏和可选的字段级敏感度。

### 5.3 从安全键盘输入

1. 用户切换到 Private Input Vault 键盘。
2. 键盘默认显示锁定状态。
3. 用户使用生物识别、设备凭据或主密码解锁。
4. 用户搜索或选择条目。
5. 用户点击要输入的具体字段。
6. 键盘只把被选中的值插入当前输入框。

### 5.4 迁移到另一台设备

1. 用户选择加密导出、本地迁移或云同步。
2. 应用打包加密后的保险库数据和元数据。
3. 新设备导入或接收加密包。
4. 用户使用主密码解锁，并核对保险库内容。

### 5.5 可选登录

1. 用户可为付费云功能创建账号。
2. 登录后启用加密云同步、设备管理和服务计费。
3. 即使用户退出登录，本地保险库功能仍可继续使用。

## 6. 功能需求

| ID | 需求 | 优先级 |

| PRD-001 | 应用必须允许用户在不登录账号的情况下创建本地保险库。 | 必须 |

| PRD-003 | 保险库必须能存储密码、用户名、备注、自定义字段和标签。 | 必须 |

| PRD-006 | 键盘默认不得提供在线词库行为。 | 必须 |
| PRD-007 | 产品必须支持加密导出和导入。 | 必须 |
| PRD-008 | 产品应该支持本地设备到设备迁移。 | 应该 |
| PRD-009 | 产品应该支持可选的第一方云同步。 | 应该 |

## 7. 平台需求

### Android

- 提供基于 Android 输入法 API 的原生应用和键盘。
- 支持本地加密存储，以及生物识别或设备凭据解锁。
- 验证键盘在普通文本框、密码框、WebView 输入框和浏览器中的行为。

### iOS

- 提供原生应用和键盘扩展。
- 将第三方键盘限制视为已知产品约束。
- 当 iOS 在敏感字段中阻止第三方键盘访问时，提供复制或导出等备选路径。
- 谨慎评估“完全访问”权限对共享保险库访问和用户信任的影响。

### HarmonyOS

- 在 API 可行性验证后，提供原生 HarmonyOS 应用和输入法集成。
- 确认打包、签名、权限、输入法注册和本地加密存储行为。

## 8. 隐私需求

- 键盘不得收集普通输入文本。
- 键盘不得上传输入文本。
- 云同步只能上传加密后的保险库载荷。
- 日志必须排除密钥、保险库内容、TOTP 种子、主密码材料和生成的验证码。

- 文档阶段之后，首个实现目标平台应选择 Android、iOS 还是 HarmonyOS？
- 用户自主控制的云路径应从加密文件导入/导出、WebDAV、S3 兼容存储，还是本地网络迁移开始？
- TOTP 是否应包含在免费离线版中，还是保留给付费层级？当前建议是将基础 TOTP 放入免费离线版。
- 第一方云是否应支持账号恢复？如果支持，哪种恢复模型可以避免服务端访问明文？
