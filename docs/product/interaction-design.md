# 核心交互设计

本文档定义 Private Input Vault 的核心交互路径。它承接 [产品决策记录：主动选择式私密输入](product-decision-record.md)、[产品整体设计](product-design.md) 和 [信息架构设计](information-architecture.md)，用于说明用户如何一步步完成关键任务，以及每条路径中的确认、错误、兜底和安全边界。

本文档不定义视觉稿、组件样式、平台代码或密码学参数。后续平台能力设计和安全实现设计应在本文档基础上继续细化。

## 1. 交互设计目标

核心交互必须强化一个产品心智：用户从本地加密保险库中主动选择要输入的私密字段，产品不替用户判断目标应用或网站是否可信。

交互设计需要达成以下目标：

- 每个敏感值插入、复制、显示前，都有清楚的用户意图。
- 用户可以在不登录、不联网时完成本地创建、保存、输入、导出和导入。
- 安全键盘保持短时输入面，不变成完整管理界面。
- 平台凭据和自动填充路径只作为兼容增强，不改变主动选择主路径。
- 关键错误尽量预防；必须提示时，提示要靠近问题、可理解、可恢复、不责备用户。
- 平台限制、锁定状态、权限缺失和导入冲突都有明确兜底路径。

## 2. 调研转译

外部调研用于校准交互原则，而不是替产品决定主路径。

### 2.1 防错优先

Nielsen Norman Group 对错误预防的建议强调：用户常在熟悉任务中发生 slip，设计应通过约束、建议、合理默认值和容错格式减少错误，而不是事后责备用户。

对本产品的转译：

- 插入敏感字段前要让用户看清字段类型和目标动作。
- 高风险动作优先使用预览、确认和可撤销策略。
- 导入和迁移必须先预览，不直接覆盖本地保险库。
- 平台限制不应显示成“用户操作错误”，而应说明当前环境不支持并给出兜底。

### 2.2 错误提示要可恢复

错误提示应靠近问题来源，使用人能理解的语言，说明发生了什么，并提供下一步动作。提示不应过早出现，也不应让用户丢失已经输入或选择的内容。

对本产品的转译：

- 解锁失败要保留用户所在任务，不把用户丢回首页。
- 导入失败要保留现有保险库，并说明是否可以重试或换文件。
- 平台填充失败要给出“打开安全键盘”“复制字段”“打开主应用”等动作。
- 表单校验应避免用户刚聚焦或探索时就显示错误。

### 2.3 表单标签必须可见

表单占位符不能替代标签。敏感字段管理尤其需要清楚、持久的标签，因为用户需要反复核对自己正在保存什么字段。

对本产品的转译：

- 条目编辑中的字段标签、类型和敏感级别必须始终可见。
- 占位文本只用于例子或补充说明，不能承载字段含义。
- 生成器、导出、导入、主密码设置都必须有清晰标签和持久说明。

### 2.4 系统认证用于可信确认

Android BiometricPrompt 和平台凭据流程强调系统级认证提示、认证类型检测和高风险操作的显式确认。iOS 自定义键盘文档强调键盘信任、最小化数据使用、必要时提供切换到其他键盘的路径，并说明第三方键盘在安全输入框等场景中的限制。

对本产品的转译：

- 主密码是根秘密，生物识别是便利解锁，不替代主密码。
- 高敏感字段显示、复制、插入可以要求重新认证。
- 系统认证提示应明确说明正在授权什么动作。
- 安全键盘必须能切换到系统键盘或其他键盘。
- iOS 等平台限制必须作为正常状态处理，而不是异常失败。

## 3. 全局交互原则

### 3.1 主动选择原则

用户必须先选择条目，再选择字段，再触发插入、复制或显示。任何自动推荐都不能跳过字段级确认。

允许的自动化：

- 根据搜索词排序结果。
- 根据收藏和最近使用排序。
- 在平台兼容路径中展示候选。
- 生成密码或 TOTP。

不允许的自动化：

- 未经用户确认自动填入敏感字段。
- 把候选排序解释为目标可信。
- 根据弱上下文静默选择某个条目。

### 3.2 最小暴露原则

每个交互只展示完成当前任务所需的信息。

- 锁定态不展示条目标题、字段标签、最近使用或搜索历史。
- 安全键盘只展示短时输入需要的信息。
- 平台兼容路径只返回系统请求范围内的必要字段。
- 错误提示不包含秘密、TOTP 种子、主密码材料或字段值。

### 3.3 可恢复原则

高风险操作必须有安全回退。

- 导入失败不覆盖现有保险库。
- 迁移中断可重新开始或继续验证。
- 平台填充失败可转安全键盘或复制。
- 解锁失败保留当前任务上下文。
- 自动锁定后可回到原任务继续。

### 3.4 离线优先原则

本地交互不得依赖账号和网络。

必须离线可用的交互：

- 创建本地保险库。
- 解锁本地保险库。
- 新建和编辑条目。
- 安全键盘搜索和字段插入。
- 密码生成和 TOTP 生成。
- 加密导出和导入。

## 4. 路径一：创建本地保险库

### 4.1 触发入口

- 首次打开主应用。
- 本地保险库不存在。
- 用户从安全键盘进入但尚未创建保险库。

### 4.2 成功路径

```text
打开应用
  -> 说明本地保险库用途
  -> 设置主密码
  -> 确认主密码
  -> 展示恢复边界
  -> 可选启用生物识别
  -> 进入空保险库
```

### 4.3 关键交互规则

- 不显示登录入口作为创建保险库前置步骤。
- 主密码设置必须说明：服务端不能恢复主密码，也不能解密保险库。
- 主密码确认不能只依赖占位符，必须有可见标签。
- 生物识别启用应写成“方便下次解锁”，不能写成“替代主密码”。
- 用户可以跳过生物识别，直接进入本地保险库。

### 4.4 错误与兜底

| 场景 | 处理 |
| --- | --- |
| 主密码太短或强度不足 | 说明风险，给出改进建议，不责备用户。 |
| 两次输入不一致 | 保留两次输入位置，提示用户重新核对。 |
| 生物识别不可用 | 允许继续使用主密码，不阻断创建。 |
| 用户中途退出 | 未完成保险库不应留下可误用的半成品状态。 |

### 4.5 退出标准

- 用户拥有一个可解锁的本地保险库。
- 用户理解主密码恢复边界。
- 创建流程没有要求登录或联网。

## 5. 路径二：保存条目和字段

### 5.1 触发入口

- 保险库空状态中的新建入口。
- 保险库列表的新建入口。
- 平台保存请求的兼容入口。
- 导入后的编辑入口。

### 5.2 成功路径

```text
点击新建条目
  -> 选择条目类型
  -> 输入标题
  -> 添加字段
  -> 设置字段类型和敏感级别
  -> 设置标签/收藏
  -> 保存到本地保险库
  -> 可选导出或同步提醒
```

### 5.3 字段编辑规则

- 字段标签、字段类型、敏感级别必须常驻可见。
- 密码、TOTP 种子、恢复码默认隐藏。
- 字段类型决定默认使用策略，但用户可以调整。
- 保存前不自动上传、不自动同步到云端，除非用户已明确启用云同步。
- 字段保存后应能立即出现在安全键盘搜索结果中。

### 5.4 字段默认策略

| 字段类型 | 默认敏感级别 | 默认行为 |
| --- | --- | --- |
| 用户名 | 普通 | 插入、复制。 |
| 密码 | 高敏感 | 插入、复制、重新认证可选。 |
| TOTP | 高敏感 | 生成临时代码，不保存生成码。 |
| 备注 | 敏感 | 显示或复制，默认不进平台填充。 |
| 地址 | 普通/敏感 | 插入、复制，适合安全键盘。 |
| 恢复码 | 高敏感 | 默认仅显示或复制，可要求重新认证。 |
| 自定义字段 | 由用户选择 | 插入、复制或仅显示。 |

### 5.5 错误与兜底

| 场景 | 处理 |
| --- | --- |
| 标题为空 | 提示需要可识别名称，可提供本地草稿名。 |
| 字段标签为空 | 给出默认标签建议，如“字段 1”。 |
| TOTP 种子格式无法识别 | 说明无法生成验证码，保留原输入供用户修改。 |
| 用户离开编辑页 | 提示保存草稿或放弃，不直接丢弃。 |
| 保存失败 | 保留用户输入，说明失败原因和重试路径。 |

### 5.6 退出标准

- 条目已本地加密保存。
- 字段策略清楚。
- 用户知道该条目可从主应用和安全键盘访问。

## 6. 路径三：安全键盘主动输入

### 6.1 触发入口

- 用户在其他应用中切换到 Private Input Vault 键盘。
- 平台填充不可用时选择安全键盘兜底。
- 用户希望输入非登录私密字段。

### 6.2 成功路径

```text
切换到安全键盘
  -> 键盘展示锁定态
  -> 用户解锁
  -> 搜索或选择收藏/最近条目
  -> 打开条目字段列表
  -> 用户点击具体字段
  -> 插入或复制字段值
  -> 清理短时敏感值
```

### 6.3 锁定态规则

- 不展示条目标题、字段标签、最近使用或搜索历史。
- 只展示产品身份、解锁入口、切换键盘入口。
- 如果本地保险库不存在，提示打开主应用创建保险库。
- 解锁失败后留在键盘任务中，不跳到无关页面。

### 6.4 解锁态规则

- 默认聚焦搜索或显示收藏/最近入口。
- 搜索只在本地保险库中进行，不上传搜索词。
- 字段列表必须展示字段标签和类型，不默认展示敏感值明文。
- 用户点击字段才触发插入或复制。
- 对高敏感字段可要求重新认证。
- 插入完成后给出轻量反馈，不显示秘密内容。

### 6.5 字段选择规则

字段按钮文案应强调动作和字段，而不是秘密值本身。

推荐结构：

```text
条目标题
  用户名    插入
  密码      插入
  TOTP      插入当前验证码
  备注      打开/复制
```

不推荐结构：

```text
直接显示密码明文
根据当前应用自动高亮“最安全”凭据
把第一个搜索结果自动插入
```

### 6.6 平台限制与兜底

| 场景 | 处理 |
| --- | --- |
| 当前输入框不接受第三方键盘 | 说明平台限制，提供打开主应用或复制路径。 |
| iOS 安全输入框替换键盘 | 在主应用和文档中说明，不把它当作应用故障。 |
| 键盘无法访问共享保险库状态 | 引导打开主应用解锁或刷新授权。 |
| 插入失败 | 保留字段选择状态，提供复制或重试。 |
| 用户需要普通输入 | 提供切换系统键盘入口。 |

### 6.7 退出标准

- 用户明确选择了字段。
- 字段值已插入或复制。
- 键盘没有收集普通键入文本。
- 敏感值已尽快清理。

## 7. 路径四：平台凭据/自动填充兼容

### 7.1 触发入口

- 系统检测到登录、密码、Passkey 或 TOTP 字段。
- 用户在系统凭据选择器中选择 Private Input Vault。
- 用户从主应用开启平台兼容设置。

### 7.2 成功路径

```text
系统请求凭据
  -> 产品接收请求
  -> 判断保险库是否锁定
  -> 如需解锁，展示认证动作
  -> 查询候选条目
  -> 用户在系统 UI 中选择候选
  -> 返回系统请求范围内的字段
  -> 清理短时授权
```

### 7.3 交互边界

- 平台候选是兼容增强，不是产品主路径。
- 用户选择候选前不返回明文字段。
- 候选匹配不能表达“目标可信”。
- 匹配不确定时应降低表达强度，例如“可能相关”而不是“推荐”。
- 用户可以关闭某个字段或条目出现在平台填充中。

### 7.4 锁定与认证

- 锁定状态只返回认证动作或空状态，不返回凭据明文。
- 认证提示应说明正在允许系统获取哪类凭据。
- 高敏感字段或异常匹配可以要求重新认证。
- 认证失败后提供安全键盘或主应用路径。

### 7.5 平台设置交互

主应用中的平台兼容状态页应提供：

- 当前平台支持能力。
- 是否已启用本产品作为键盘。
- 是否已启用自动填充或凭据提供方。
- 支持密码、Passkey、TOTP 的状态。
- 系统设置入口。
- 失败时的兜底路径。

### 7.6 错误与兜底

| 场景 | 处理 |
| --- | --- |
| 系统未启用本产品为凭据提供方 | 展示系统设置引导，不要求登录。 |
| 请求类型不支持 | 说明当前字段不支持平台填充，提供安全键盘。 |
| 没有匹配候选 | 提供搜索保险库、打开主应用或安全键盘。 |
| 浏览器/应用上下文不完整 | 降低匹配置信，不暗示安全。 |
| 用户取消认证 | 返回原输入场景，不删除本地状态。 |

### 7.7 退出标准

- 平台只获得用户选择范围内的数据。
- 本地保险库仍可独立使用。
- 平台失败时存在主动输入兜底。

## 8. 路径五：加密导出与导入

### 8.1 导出触发入口

- 备份与迁移区域。
- 版本发布前备份提醒。
- 用户准备换设备。

### 8.2 导出成功路径

```text
进入备份与迁移
  -> 选择加密导出
  -> 说明导出内容和风险
  -> 用户确认
  -> 重新认证
  -> 创建加密导出包
  -> 用户选择保存位置
  -> 展示导出完成和恢复说明
```

### 8.3 导出规则

- 导出包必须加密和认证。
- 导出前说明：拿到导出包不等于能读明文，仍需要主密码或恢复材料。
- 导出位置由用户选择。
- 导出过程不上传到第一方云，除非用户明确选择云路径。
- 导出完成后不展示秘密内容。

### 8.4 导入触发入口

- 备份与迁移区域。
- 首次启动时选择从备份恢复。
- 本地迁移接收端。

### 8.5 导入成功路径

```text
选择导入
  -> 选择加密包
  -> 验证包格式和认证信息
  -> 输入主密码或恢复材料
  -> 展示导入预览
  -> 用户选择导入策略
  -> 应用导入
  -> 展示结果和冲突处理
```

### 8.6 导入策略

| 策略 | 用途 | 规则 |
| --- | --- | --- |
| 新建保险库 | 当前无本地保险库。 | 导入后成为本地保险库。 |
| 合并 | 当前已有保险库。 | 只新增或更新用户确认的条目。 |
| 替换 | 用户明确要覆盖。 | 必须二次确认，并保留回滚机会。 |
| 预览 | 用户只想检查内容。 | 不写入本地保险库。 |

### 8.7 错误与兜底

| 场景 | 处理 |
| --- | --- |
| 文件格式不支持 | 说明不是有效导出包，允许重新选择。 |
| 认证失败或被篡改 | 拒绝导入，不修改现有保险库。 |
| 主密码错误 | 保留导入上下文，允许重试。 |
| 版本不兼容 | 说明需要更新应用或选择兼容导入。 |
| 冲突过多 | 提供逐项选择和取消导入。 |

### 8.8 退出标准

- 导出包已安全生成，或导入结果已明确展示。
- 失败不会破坏现有保险库。
- 用户知道下一步如何恢复或迁移。

## 9. 跨路径状态规则

### 9.1 自动锁定

自动锁定可能发生在主应用、安全键盘或平台兼容路径中。

规则：

- 自动锁定后不丢失未保存编辑内容。
- 解锁后回到原任务上下文。
- 高敏感字段短时值必须清理。
- 锁定原因不应暴露秘密。

### 9.2 重新认证

重新认证用于高敏感操作。

可触发场景：

- 显示密码明文。
- 复制密码、TOTP、恢复码。
- 插入高敏感字段。
- 导出保险库。
- 替换本地保险库。
- 修改主密码或安全设置。

### 9.3 剪贴板

剪贴板是兜底能力，不是默认主路径。

规则：

- 复制前明确字段和风险。
- 支持自动清除。
- 平台不允许清除时说明限制。
- 不把复制内容写入日志或诊断。

### 9.4 平台限制

平台限制应被当作正常状态处理。

表达方式：

- “当前平台不允许在此输入框使用第三方键盘。”
- “当前应用没有提供可用的自动填充上下文。”
- “可以改用安全键盘、复制字段或打开主应用。”

避免表达：

- “操作失败。”
- “输入错误。”
- “无法识别安全网站。”

## 10. 交互文案原则

### 10.1 标签

- 字段标签必须常驻可见。
- 占位符只作为示例，不替代标签。
- 高敏感字段要标明类型，不默认展示值。

### 10.2 确认

确认文案必须说明动作后果。

示例：

- “导出加密保险库。”
- “替换当前本地保险库。”
- “显示此恢复码。”
- “复制密码，稍后自动清除剪贴板。”

### 10.3 错误

错误文案应包含三部分：发生了什么、为什么重要、下一步做什么。

示例：

- “导入包验证失败。文件可能已损坏或被修改。当前保险库没有变化，请重新选择文件。”
- “当前输入框不允许第三方键盘。可以打开主应用复制字段，或切换回系统键盘。”
- “主密码不匹配。请重新输入；本地保险库没有被修改。”

## 11. 验收清单

后续线框、原型和平台实现应使用以下清单验证：

- 用户是否明确选择了条目和字段？
- 插入、复制、显示是否都有清楚动作？
- 高敏感字段是否有重新认证策略？
- 创建、保存、输入、导出和导入是否离线可用？
- 安全键盘是否保持短时输入面？
- 平台填充是否被描述为兼容增强，而不是主路径？
- 错误提示是否靠近问题、能恢复、不责备用户？
- 表单标签是否始终可见？
- 平台限制是否有安全键盘、复制或主应用兜底？
- 导入失败是否不破坏现有保险库？
