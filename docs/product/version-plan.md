# 版本与权益计划

本文档用于区分发布阶段和商业版本。

## 1. 发布阶段

### v0.1 可行性原型

目标：证明产品可以作为由用户主动选择内容的安全键盘运行。

- Android 键盘原型。
- 本地保险库模拟实现，或最小可用的加密保险库。
- 手动搜索条目并插入字段。
- 支持基础密码、用户名、自定义字段和备注。
- 记录 iOS 和 HarmonyOS 的平台限制。

退出标准：

- 键盘可以把用户选择的值插入常见 Android 文本字段。
- 密码字段和浏览器字段已完成测试并形成文档。
- 原型路径不需要任何网络调用。

### v0.2 本地保险库 MVP

目标：交付第一个真正可用的离线产品。

- 本地加密保险库。
- 主密码设置。
- 在可用平台上支持生物识别或设备凭据解锁。
- 条目增删改查。
- 安全键盘字段选择。
- 密码生成器。
- 基础 TOTP 生成。
- 加密导出/导入。
- 带自动清除的剪贴板备选方案。

退出标准：

- 用户可以离线创建、使用、导出和恢复保险库。
- 敏感字段不会被写入日志。
- 核心加密测试和保险库迁移测试通过。

### v0.3 跨设备迁移

目标：支持用户自主控制的数据迁移，不要求云账号。

- 加密迁移包。
- 用于本地迁移的二维码或手动配对设计。
- 迁移校验和可恢复导入。
- 导入保险库版本的冲突处理。
- 评估桌面或 CLI 打包辅助工具。

退出标准：

- 用户无需第一方云即可把保险库迁移到另一台设备。
- 迁移包已加密并通过认证保护。
- 导入失败不会破坏现有保险库。

### v1.0 公开发布

目标：在首个受支持平台上提供稳定的离线优先产品，并清楚说明平台特定限制。

- 生产级本地保险库。
- 生产级安全键盘。
- 基础 TOTP。
- 加密备份/恢复。
- 清晰的隐私和安全文档。
- 发布测试套件和手动 QA 清单。

退出标准：

- 发布质量门禁通过。
- 平台限制已在产品内和发布说明中记录。
- 核心本地使用不需要登录。

### v1.1 可选云同步

目标：增加基于付费和登录的云服务，同时不削弱离线优先行为。

- 用户账号。
- 端到端加密云同步。
- 设备列表。
- 同步冲突处理。
- 版本历史和恢复。
- 订阅与权益检查。

退出标准：

- 云端只存储密文。
- 用户退出登录或离线时，本地保险库仍可使用。
- 同步失败不会阻断本地使用。

## 2. 商业版本

### 免费离线版

不需要登录。

- 本地加密保险库。
- 安全键盘。
- 密码生成器。
- 自定义字段和标签。
- 基础 TOTP。
- 加密导出/导入。
- 带自动清除的剪贴板备选方案。
- 通过加密包手动迁移。

### Plus

仅服务功能需要登录。

- 第一方端到端加密云同步。
- 多设备同步。
- 设备管理。
- 版本历史。
- 云备份恢复。
- 如果后续增加附件，可提供更高备份容量上限。

### Pro

权益校验和可选服务功能需要登录。

- 多保险库。
- 私密字段高级模板。
- 高级迁移工具。
- 安全报告。
- 如果商业策略决定限制，可提供自定义存储连接器。
- 优先支持。

### 家庭或团队版

需要登录。

- 共享保险库。
- 成员和权限管理。
- 如果安全恢复模型获批，可提供恢复联系人或紧急访问。
- 便于审计、但不包含秘密内容的活动元数据。

### 外观与附加（可选付费，不影响功能）

- 键盘皮肤等外观个性化。
- 其他不影响正常功能的可选项。
- 这类增值纯属锦上添花：不获取、不解密任何保险库内容，关闭或未购买都不影响密码、2FA 等正常使用。

## 3. 登录策略

- 创建、解锁、搜索、编辑、导出或使用本地保险库时，绝不能要求登录。
- 第一方云同步、订阅、设备管理、家庭/团队共享和付费服务支持可以要求登录。
- 对于仅本地生效的付费功能，应在可行时支持离线许可证文件或平台商店权益，但首个版本不强制要求。

## 4. 功能分级原则

- 核心隐私和本地安全功能保持免费。
- 有服务端成本的功能可以要求登录和订阅。
- 帮助用户迁出或备份数据的功能，不应被完全锁定在第一方云之后。
- 增值业务（云同步多端共享、键盘皮肤等外观）必须账号/会员绑定、可延后于 MVP，且绝不影响或降级免费的正常使用。
- 免费层始终保证：密码管理、2FA/TOTP、本地加密保险库、加密导出与手动迁移可离线使用，无需登录。
- 任何付费功能都必须保留“服务端无法解密保险库内容”的承诺。
