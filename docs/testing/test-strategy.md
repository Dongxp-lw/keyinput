# 测试策略

## 1. 测试目标

- 证明核心离线使用不依赖登录和网络。
- 证明敏感值在静态存储时加密，且不会进入日志。
- 证明键盘只插入用户选择的字段。
- 证明导出/导入和迁移流程不会损坏保险库数据。
- 证明可选云同步只存储密文，且不阻断本地使用。
- 在发布前记录平台限制。

## 2. 测试层级

### 核心单元测试

- 条目模型校验。
- 字段敏感度规则。
- 使用已知测试向量验证 TOTP 生成。
- 密码生成器约束。
- 保险库序列化和反序列化。
- 架构迁移行为。

### 加密与保险库测试

- KDF 参数校验。
- 认证加密往返测试。
- 保险库文件篡改检测。
- 导出包篡改检测。
- 错误密码失败行为。
- 向后兼容迁移夹具。

### 平台集成测试

- Android 键盘向普通字段、密码字段、浏览器和 WebView 插入内容。
- iOS 键盘扩展在普通字段和受限制敏感字段中的行为。
- API 验证后的 HarmonyOS 输入法行为。
- 生物识别解锁和备选流程。
- 应用到键盘的共享状态行为。

### 隐私测试

- 离线流程中不产生网络调用。
- 日志中不包含敏感值。
- 崩溃报告中不包含敏感值。
- 键盘不收集普通输入文本。
- 在可行时保护截图和应用切换预览。

### 端到端测试

- 离线创建本地保险库。
- 添加密码条目和自定义字段。
- 解锁键盘并插入选中字段。
- 生成并输入 TOTP 验证码。
- 导出加密保险库。
- 在另一个干净配置中导入加密保险库。
- 在曾经登录后退出登录的状态下继续使用应用。

### 云测试

云测试只在云支持实现后适用。

- 上传加密同步对象。
- 在第二台设备下载并解密。
- 确认服务端在正常运行中无法访问明文。
- 测试同步冲突解决。
- 测试离线编辑后稍后同步。
- 测试退出登录不会删除本地保险库，除非用户明确选择该操作。

## 3. 手动 QA 重点

- 键盘解锁阻力。
- 键盘内搜索速度。
- 防止误插入。
- 对 iOS 限制的清晰提示。
- 备份和恢复可信度。
- 忘记主密码时的恢复警告。

## 4. 发布质量门禁

只有满足以下条件，版本才能发布：

- 核心单元测试通过。
- 加密与保险库篡改测试通过。
- 网络关闭时离线使用测试通过。
- 支持的平台版本上，键盘插入手动 QA 通过。
- 导出/导入往返测试通过。
- 已知日志路径不会打印秘密。
- 平台限制已形成文档。
- 任何云功能或恢复行为都通过安全评审。

## 5. 测试环境

- 本地开发机器。
- Android 模拟器和 Android 真机。
- iOS 模拟器和 iPhone 真机。
- 可用时使用 HarmonyOS 模拟器和真机。
- 用于离线验证的隔离无网络测试环境。
- 用于可选同步功能的测试云环境。
