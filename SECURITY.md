# 安全策略（Security Policy）

Private Input Vault 是一个**以安全为第一要务**的离线加密保险库与私密输入键盘。我们认真对待每一个安全问题。

## 支持的版本

项目处于早期开发阶段。安全修复仅针对**最新开发版（`main`）与最近一次发布**提供。

| 版本 | 是否支持安全修复 |
| --- | --- |
| `main`（开发版） | ✅ |
| 最近一次 release | ✅ |
| 更早版本 | ❌ |

## 如何报告漏洞

**请不要在公开的 Issue、Pull Request、Discussions 或社交渠道披露安全漏洞。**

> 所有公开联系方式集中在 [项目配置](project-config.toml) 的 `[contact]` / `[repository]`，发布前在那里统一填写。当前为**单人本地开发、尚未设远端**，下列渠道待选定后启用：

1. **私有安全通报**：若采用 GitHub/GitLab，启用其「私有漏洞报告（Private vulnerability reporting）」并把入口链接登记到项目配置。
2. **加密邮件**：使用项目配置 `[contact].security_email`（如有 PGP，公钥指纹登记在 `security_pgp`）。

报告请尽量包含：

- 受影响的组件/版本（核心、App、键盘 IME 等）与复现环境；
- 复现步骤或 PoC；
- 影响评估（能拿到什么、前置条件）；
- 可选的修复建议。

> ⚠️ 报告中**请勿**附带任何真实主密码、密钥或他人真实数据。

## 我们的响应承诺

- **3 个工作日**内确认收到。
- **10 个工作日**内给出初步评估与处理计划。
- 修复发布后，在你同意的前提下进行**致谢**（见下）。
- 我们遵循**协调披露**：修复就绪并发布后再公开细节。

## 范围

属于安全问题（欢迎报告）：

- 绕过加密、主密码、解锁或自动锁定；
- 主密码/密钥/字段值/TOTP 种子或码**泄漏到日志、剪贴板、崩溃报告、备份明文或网络**；
- 导出包/迁移包的机密性或完整性问题；
- 安全键盘把秘密泄漏给非目标输入框或系统剪贴板；
- 权限提升、可被利用的内存安全问题。

通常**不**单独计为漏洞（但仍欢迎反馈）：

- 需要已 root/已越狱设备或物理拆机离线暴力破解（在威胁模型之外，见 [安全模型](docs/technical/security-model.md)）；
- 缺少纵深防御加固项（如代码混淆）——属增强项，按威胁模型决定。

## 我们的安全立场（已实现/设计）

- **离线优先**：核心不声明 `INTERNET` 权限，明文不离开设备。
- **强加密**：Argon2id + XChaCha20-Poly1305 + HKDF；不自实现密码学原语，仅用审计过的库。
- **零日志**：秘密绝不进日志/崩溃报告（多层强制 + CI 门禁）。
- 详见 [安全模型](docs/technical/security-model.md) 与 [安全实现设计](docs/technical/security-implementation-design.md)。
