# Agent 工作区

本文件夹是负责开发的 AI agent 的工作区与"执行大脑"。因为 agent 的上下文记忆有限，这里持久化保存**计划、任务、状态、决策**，让任何一次新会话都能无缝接续，不依赖易失的对话记忆。

## 为什么有这个文件夹

- 用户不做移动端开发，开发由 agent 负责；需要可追踪、可接续的执行体系。
- 自顶向下、一层一层地落实架构，避免项目失控。
- 任务粒度要足够小，每个任务可独立完成并验证。

## 文件职责（单一事实来源）

- [execution-plan.md](execution-plan.md)：自顶向下的总执行计划（分层路线图、顺序、各层门禁、与版本计划的映射、工作方法）。**改"怎么走"看这里。**
- [tasks.md](tasks.md)：细粒度任务板（任务 ID、所属层、状态、依赖、验收证据）。**改"做什么"看这里。**
- [state.md](state.md)：当前进度与下一步动作（构建阶段的权威状态）。**接续工作先读这里。**
- [decisions.md](decisions.md)：已锁定/待定的关键决策登记（ADR-lite）。**避免反复推翻决策。**
- `notes/`：按需创建的工作笔记与调研草稿。

## 与现有文档的关系

- `docs/` 是项目的权威文档（产品/技术/实现）。"完善架构"= 丰富 `docs/technical/`，不放这里。
- `.agent/` 只放 agent 的执行管理与工作记忆，不复制 `docs/` 的结论，只引用与链接。

## 纪律

- 每完成一个任务：更新 [tasks.md](tasks.md) 状态 + [state.md](state.md) 的"下一步"，必要时在 [decisions.md](decisions.md) 记决策。
- 不臆造平台能力；未验证标"待验证"。
- 写最终生产代码前，先把对应层的计划与任务做实。
