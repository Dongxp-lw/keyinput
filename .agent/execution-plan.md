# 总执行计划（自顶向下，分层推进）

本计划把"完整的设计文档"推进到"可构建、组织良好的移动应用"。原则：从最高层架构出发，一层一层向下细化，每层有明确产出与门禁；上层不稳定不进入下层。

服从 [产品决策记录](../docs/product/product-decision-record.md) 基线；技术依据见 [技术架构](../docs/technical/architecture.md) 与 [安全实现设计](../docs/technical/security-implementation-design.md)。

## 0. 现状与目标

- 现状：设计文档完整（产品/技术/版本/测试/11 份实现文档 + 开源组件选型）；**无任何代码或工程骨架**（`apps/`、`core/`、`tools/` 仅占位 README）。
- 目标：在 Android 优先前提下，按版本计划逐步构建真实应用；开发由 agent 负责，用户审阅与支持。
- 工作方式：本计划分层；每层细化为 [tasks.md](tasks.md) 的小任务；进度记 [state.md](state.md)；决策记 [decisions.md](decisions.md)。

## 1. 分层模型（顶 → 底）

| 层 | 名称 | 产出 | 主要依据 | 对应版本 |
| --- | --- | --- | --- | --- |
| L0 | 工程基础（骨架） | 可编译的空骨架：模块结构、Gradle、Kotlin、SDK、约定、CI 雏形 | 技术架构 §1-2 | v0.1 |
| L1 | 架构落地 | 模块边界与包结构、层间公共接口（契约）、依赖方向规则、共享核心语言决策 | 技术架构 §1-3、各实现文档接口 | v0.1 |
| L2 | 横切基础 | 加密/序列化/安全存储/日志卫生/错误模型/CSPRNG 的库选定与薄封装 | 安全实现设计 §2-4、§7、§2.4；架构 §2 选型 | v0.2 |
| L3 | 领域核心（纯 Kotlin，可测） | ENTRY、VAULT、LOCK、GEN、TOTP、IMEX 的核心逻辑（无平台 UI） | 对应实现文档 + 其 AI 任务 | v0.2 |
| L4 | 平台集成（Android） | 安全键盘(KBD)、生物识别(BIO)、剪贴板(CLIP)、主应用 UI、持久化接线 | KBD/BIO/CLIP 实现文档 | v0.2 |
| L5 | 跨设备与云 | MIGR（迁移）、SYNC（云同步） | MIGR/SYNC 实现文档 | v0.3 / v1.1 |
| L6 | 测试与发布门禁 | TP-* 测试、MASVS 门禁、离线与隐私验证 | 测试计划、v1.0 发布计划 | 各版本 |

依赖方向：平台层只依赖核心层，核心层不反向依赖平台。

## 2. 执行顺序与里程碑

1. 设立执行体系（本文件夹）——进行中。
2. L0 工程基础：先在 `docs/technical/` 产出"模块结构与构建"细化，再落地空骨架。
3. L1 架构落地：把实现文档里的 Kotlin 接口提为模块间契约。
4. L2 → L3 → L4：按 v0.2 MVP 顺序构建横切基础、领域核心与 Android 集成。
5. L5/L6 随 v0.3、v1.0、v1.1 推进。

里程碑对齐版本计划 v0.1 → v1.1（见 [版本与权益计划](../docs/product/version-plan.md)）。

## 3. 各层门禁（达标才进下层）

- L0：骨架可 `./gradlew assemble` 通过；模块与包结构经评审。
- L1：层间接口（契约）确定且核心模块不依赖平台。
- L2：加密/序列化库与版本锁定，已知答案测试通过。
- L3：领域核心单元测试通过，不记录秘密。
- L4：键盘可从真实保险库主动选择插入；生物识别可解锁可回退。
- L5/L6：对应 TP-* 与发布门禁通过。

## 4. 工作方法（针对 agent 短记忆）

- 单一事实来源：计划在本文件，任务在 [tasks.md](tasks.md)，状态在 [state.md](state.md)，决策在 [decisions.md](decisions.md)。
- 任务粒度：每个任务可一次完成并独立验证，自带验收证据，不依赖隐含上下文。
- 每次会话开始：读 [state.md](state.md) → 选 [tasks.md](tasks.md) 中下一个就绪任务 → 做 → 更新状态。
- 验证纪律：沿用文档阶段检查（诊断、尾随空白、链接、基线措辞），代码阶段加构建与测试。
- 决策不反复：已锁定决策进 [decisions.md](decisions.md)；要改先在那里改并记原因。
